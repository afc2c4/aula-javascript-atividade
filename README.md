# Atividade Prática: Corrigindo uma Landing Page

## Objetivo

Esta atividade foi criada para que os alunos revisem e corrijam uma landing page feita com HTML5, CSS3, Bootstrap e JavaScript.

O tema da página é um curso de Desenvolvimento de Testes.

O projeto foi montado com erros comuns cometidos por desenvolvedores júnior. A tarefa da turma é corrigir esses problemas até que a página fique funcional, navegável e executável no navegador.

## Arquivos do projeto

- index.html
- styles.css
- script.js

## O que precisa ser corrigido

- Ajustar a referência dos arquivos locais carregados pelo HTML.
- Corrigir problemas de navegação entre as seções da página.
- Corrigir erros de integração com componentes do Bootstrap.
- Revisar classes do grid e de layout que estão impedindo a responsividade esperada.
- Corrigir regras CSS inválidas ou escritas incorretamente.
- Revisar identificadores usados no HTML e no JavaScript.
- Corrigir erros no envio do formulário.
- Ajustar a exibição da mensagem de retorno ao usuário.

## Requisitos da entrega

- A landing page deve continuar com 6 seções principais.
- O layout deve funcionar em desktop e mobile.
- O menu deve navegar corretamente para as seções.
- O formulário deve funcionar sem erro no navegador.
- Os estilos locais devem ser aplicados corretamente.

## Execução

Abra o arquivo index.html no navegador após realizar as correções.
