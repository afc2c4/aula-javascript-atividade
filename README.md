# Atividade Pratica: Debug de Landing Page (HTML5, CSS3, Bootstrap e JavaScript)

## Contexto

Voce recebeu uma landing page extensa de um evento de tecnologia.
O projeto foi entregue com erros intencionais de:

- Sintaxe
- Semantica
- Integracao com Bootstrap
- Usabilidade e experiencia do usuario

A proposta e corrigir tudo para deixar a pagina funcional, responsiva e compreensivel para o usuario final.

## Arquivos da atividade

- index.html
- styles.css
- script.js

## Tipos de erro que foram colocados no projeto

### HTML / Semantica

- Caminhos de arquivos incorretos
- IDs e anchors inconsistentes
- Relacoes erradas entre label e input
- Elementos com semantica inadequada
- Links de navegacao quebrados

### CSS / Layout

- Propriedades escritas errado
- Regras com unidade faltando
- Media query invalida
- Estilos importantes nao aplicados por erro de sintaxe

### JavaScript / Comportamento

- Seletores de elementos incorretos
- Eventos registrados com nome errado
- Erros de sintaxe no codigo
- Validacoes inconsistentes
- Persistencia no localStorage com chave/valor incorretos

## Objetivo da correcao

Ao final, a pagina deve:

- Carregar CSS e JS corretamente
- Ter menu funcional com navegacao para secoes validas
- Manter semantica HTML adequada (estrutura, labels, ids, hierarquia de titulos)
- Funcionar no Bootstrap sem erros de integracao
- Ser responsiva em desktop e mobile
- Permitir envio de formulario com validacao coerente
- Exibir mensagens de sucesso/erro corretamente para o usuario
- Nao apresentar erro de sintaxe no console

## O que deve constar no .md final da correcao (EXPECTATIVA)

Crie um arquivo de entrega em Markdown (exemplo: CORRECAO.md) contendo, no minimo:

1. Resumo da abordagem
- Breve descricao de como voce investigou e corrigiu os erros.

2. Lista de erros encontrados e correcao aplicada
- Erro
- Causa
- Correcao feita
- Arquivo afetado

3. Evidencias de funcionamento
- Menu navegando entre secoes
- Formulario validando e exibindo feedback
- Layout responsivo funcionando
- Console sem erros

4. Testes manuais executados
- Passo a passo do que foi testado
- Resultado esperado vs resultado obtido

5. Melhorias extras (opcional)
- Ajustes de UX, acessibilidade e organizacao de codigo

## Criterios de avaliacao sugeridos

- Correcao tecnica dos erros
- Qualidade da semantica HTML
- Qualidade visual e responsividade
- Confiabilidade do JavaScript
- Clareza do relatorio final em Markdown

## Execucao

1. Abra o index.html no navegador.
2. Corrija os arquivos index.html, styles.css e script.js.
3. Recarregue e valide o comportamento da pagina.
4. Gere o arquivo .md com a entrega final seguindo a expectativa acima.
